## 🌍 背景

2026 年世界杯是这项赛事**历史上最大的一次扩张**：

- **48 队**（从 32 队扩），**12 个组**（从 8 组），**104 场比赛**（从 64 场）
- **三个东道主**：🇺🇸 美国 · 🇨🇦 加拿大 · 🇲🇽 墨西哥（首次三国合办）
- **16 座主办城市**：11 个美国城市 + 2 个加拿大城市 + 3 个墨西哥城市
- **比赛窗口**：2026/6/11 揭幕战 — 7/19 决赛（跨 5 周）
- **新赛制**：12 组各 4 队，前 2 + 8 个成绩最好的第 3 名 → **32 强**进入 R32，再打 R16 → QF → SF → 决赛（多了一轮 R32）

**为什么写这个 skill？**

1. **赛事太密**：104 场 = 每天平均 3 场，光看赛历根本选不出哪些值得看
2. **时区坑**：大部分比赛在美东时间白天 → 北京时间凌晨，老球迷要「熬夜清单」不要「全部」
3. **强强队太多**：扩军后 S/A 级球队密度更高，FIFA 排名和实际观赛价值不对等
4. **恩怨局冷门**：很多人不知道巴西-苏格兰（1990 邓加首秀）、哥伦比亚-葡萄牙（2016 小组）这种历史
5. **赔率有价值**：扩军后冷门更多，下位球队（赔率 ≥8）反而有更高爆冷空间

**这个 skill 解决什么？**

把 104 场比赛按 **强度分（0-100）** 排序，自动过滤「菜鸡互啄」和「无噱头对决」，让你 5 秒决定今晚看不看。

### 三种模式

#### 1️⃣ 球队模式
**输入**：队名（中文 / 英文 / key 都可以）
**输出**：该队全部比赛 + 强度分 + 恩怨标签
**例**：「葡萄牙的赛程」 / 「Brazil schedule」 / 「J 组赛程」

#### 2️⃣ 强度模式（核心）
**输入**：阈值（默认 70）+ 开关（关菜鸡 / 关无噱头）
**输出**：按强度降序的比赛列表（最多 15 行）
**例**：「熬夜清单」 / 「强度 ≥65」 / 「关掉菜鸡」

**强度分公式**（0-100）：
```
strengthBase   = 100 × (1 - (fifaRank_a + fifaRank_b) / 200)         # 0~100
stageBonus     = {group:0, R32:3, R16:8, QF:15, SF:25, F:40}
rivalryBonus   = 10 if (a,b) 在恩怨清单
starBonus      = min(8, (trio.boost 总和) × 0.75)                   # 0~8
intensityScore = min(100, strengthBase + stageBonus + rivalryBonus + starBonus)
```

**三档推荐**：
- **≥70** 激情 — 真球迷，熬就熬了
- **55-69** 值得看 — 周末想看但不熬
- **<55** 随便看看 — 默认全开

#### 3️⃣ 赔率模式
**输入**：最低下位赔率
**输出**：高赔率冷门 + 爆冷空间
**例**：「赔率 ≥8 的冷门」 / 「找冷门」

## 🗃️ 数据 schema

### `matches.json`

```json
{
  "_meta": { "source", "matchesCount", "groupCount", "teamCount", "format" },
  "groups": { "A": ["mexico", "south_korea", "czech_republic", "south_africa"] },
  "venues": { "TBD": { "city", "stadium", "tz", "country" } },
  "matches": [
    {
      "id": "J3",                          // 组+序号 或 R32-N / R16-N / QF-N / SF-N / 3RD / F
      "stage": "group",                    // group | R32 | R16 | QF | SF | 3rd | F
      "group": "J",                        // group 阶段才有效
      "date": "2026-06-23",                // BJT 日期
      "time": "01:00",                     // BJT 时间
      "utcDate": "2026-06-22",             // UTC 日期
      "utcTime": "17:00",                  // UTC 时间
      "home": "argentina",
      "away": "austria",
      "venue": "TBD"
    }
  ]
}
```

### `teams.json`

```json
{
  "argentina": {
    "name": "Argentina",
    "nameCN": "阿根廷",
    "fifaRank": 1,
    "tier": "S",                          // S | A | B | C | D
    "group": "J",
    "oddsWin": 5,                          // 夺冠赔率
    "oddsTopScorer": 5.5,
    "notes": "Defending champions, Messi's last dance"
  }
}
```

**tier 含义**：
- `S` — 争冠（FIFA top 8）
- `A` — 强队，淘汰赛常客
- `B` — 中游，可爆冷
- `C` — 下狗
- `D` — 陪跑，小组出局大概率

### `rivalries.json`

```json
{
  "rivalries": [
    { "pair": ["argentina", "france"], "name": "2022 Final Replay", "type": "recent_final" }
  ],
  "star_attractions": {
    "trios": [
      { "tag": "Messi farewell", "team": "argentina", "weight": 8 }
    ]
  }
}
```

**恩怨类型**：
- `historic` — 历史宿敌（阿-英、德-荷、巴-阿）
- `recent_final` — 近期决赛/半决赛
- `regional` — 同地区德比
- `political` — 政治敏感
- `star_duel` — 球星对决

## 🔄 如何更新数据

### 小组赛（72 场）
已基本稳定，开赛不再变。直接改 `assets/matches.json` 的 `matches[]` 数组里对应 ID。

### 淘汰赛（32 场）
目前是 TBD 模板。**每打完一轮**就说「更新淘汰赛 / 填入晋级队」，AI 引导你提供晋级结果，写回 `matches.json` 的 R32-N / R16-N / QF-N / SF-N / 3RD / F 对应项的 home/away 字段。

### 赔率 / 排名
说「更新赔率」/「更新排名」，贴一份最新盘口或 FIFA 排名 JSON，AI 按 teamKey 覆盖到 `teams.json`。

### 恩怨清单
直接编辑 `assets/rivalries.json` 的 `rivalries[]`，加新条目即可。`pair` 数组里两个 key 顺序无所谓，AI 会按字母排序后查表。

## ⚠️ 已知限制

1. **数据是预热快照**：FIFA 排名和赔率基于 2026 年初共识，开赛后可能漂移。说「更新赔率」就能刷新
2. **没有 STT**：语音消息听不到，打字就好
3. **BJT 主显示**：默认北京时间，国际用户需要 +8 倒推
4. **淘汰赛 R32 之后无对手**：要等小组赛打出来才能填，对应场次现在强度只是 base（无 stageBonus）
5. **强度公式不完美**：FIFA 排名是滞后的（年初/年末发布），强队状态起伏（新教练、伤病）会失真
6. **球场信息 TBD**：开赛前 FIFA 才会公布每场具体球场
